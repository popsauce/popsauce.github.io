Introduction

I started getting piqued by the idea of using a functional programming language after coming across a lot of posts on lisp. And its position on the language power continuum.

Even on developer surveys 2 functional languages were number 2 and 3 on this list as being the most loved languages.

Now that might not indicate too much cause at the end of the day its a tool to get things done and its definitely better to use a language more suited to the task at hand.

But all in all it had me convinced that i could not ignore this mystery paradigm anymore.

When it comes to deciding on a language i think theres a spectrum to follow some things require speed, some things require existing libraries that are required for a particular kind of task, some things are easier in one language harder in another and all other possible variables you can think of like good forums and good resources etc that could dictate how much leverage you might get when using that language.

So after making my case for dabbling into a functional programming language i had to decide on one, or maybe a few ?

Some of those choices were :-

  • Haskell for the Best Possible Implementation of FP in a Language
  • Scala, Clojure for Apps That Can Run On JVM
  • Elixir, Erlang for Applications That Run on BeamVM, for Scalable Fault Tolerant Highly Available Systems
  • F# for Apps that run on the .NET framework
  • Common Lisp, for Lisp macros
  • Clojurescript, Purescript as an Alternative for JavaScript

After trying to compare by going through lots of posts on all possible aspects it seemed that there is no golden bullet.

From extensive comparisons between functional and non functional languages.

To comparing a single metric.

This rabbithole had no end.

Dabbling into Functional Programming

So now that I had a general direction about which langauges I wanted to try, to learn FP.

I thought of solving a set of problems. Problems that arent too hard yet arent too easy, just enough so that i can get a general idea about how to do things the FP way.

The problems are :

  • Given a set of x,y points we have to determine whether it lies on any of a given set of general functions if it doesnt we return “null”
  • Take a set of strings encode them into a single string and then decode them again.
  • Implement a fast access table similar to a hash table/dictionary

In case youre wondering where these problems came from, I just made them up.

And we’ll try to solve these problems using these languages

  • Haskell
  • Elixir

This choice i think could be attributed to chance, its just that after going through so many posts i just gravitated more towards these two as my first candidates, but i’ll probably stick with haskell given the fact that it is a better language to grasp functional programming concepts. I might transition to another language later but for now im just trying out these 2.

Determine if a set of points lie on a general function in the xy plane.

For this problem our input is straightforward we take in a set of points [$(x_{1},y_{1})(x_{2},y_{2})(x_{3},y_{3})$]. And check if they lie on either of these general functions : $$ 10^x, e^x, x^2, x, \sqrt{x}, \ln(x), \log_{10}(x) $$

And return the function on which the point lies, if it doesn’t we return null.
