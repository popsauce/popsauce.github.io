I was on the lookout for a custom object detection solution for 2 projects that i had in mind. Mostly to keep track of certain items and do some estimation based tasks like inventory management and update their status in real time.

For example a scenario where a desk is in the field of view of the camera it should be able to detect objects present on the desk and then update a table of of values that maintain things present on the desk.

So starting with some basic design considerations we need the following things more or less :

  • Portable camera for capturing a video stream for object detection
  • Real time camera streaming (good to have)
  • Adequate compute for object detection
  • A way to interface with camera and schedule cron jobs if possible
  • Extracting information from the video scene and storing it in a csv file containing the updated values of the last streaming session.

Potential options for a portable camera

Camera module is probably the most important thing in this project so here are the potential options when it comes to digital cameras -

  • camera based SoC / microcontroller
  • webcameras
  • consumer grade portable digital cameras like gopros
  • smartphone cameras

What varies wildly amongst all these offerings is their cost, the size of their image sensors, thier recording resolution ranges and all these other variables

I wont mention all the options that i went through but some of the major ones that stood out for my use case were esp32 based cameras, stm32f4 /f7 boards with openvision camera modules, raspberry pi soc with camera modules.

And i even considered webcameras and old smartphone cameras cause i think its much better to go for an old smartphone camera as its the best quality camera you can get on a budget but still i wanted to test an soc based solution so i mostly went with using an old smartphone camera for testing the setup and a raspberry pi 0 2 w board for my image processing needs.

Some options that i was considering at the time are listed here :-

  • Esp 32 with openvision camera modules like ov2640, 5640, 7670, 9655.
  • Raspberry Pi modules ranging from Raspberry Pi Zero 2 W to Raspberry Pi 4 with their respective camera modules.

The sections below detail how to setup both types of cameras

Setting up the RPi 0 2 W

So as a potential portable battery powered camera this was a good choice compared to a smartphone as it costs $5 for microsd card, $15 for the rpi 0 2 w, $15 for the v2.1 camera module and $15-30 for a powerbank (10 Ah to 20 Ah) so around $50. An old smartphone camera might cost around this much if not more like an old pixel camera or and old iphone.

Prerequisites

For this setup im using the 64bit Bullseye lite os based on debian 11, while flashing the image it is necessary to use the raspi-imager provided by rpi especialy for this headless setup as that is the only way to enable and run ssh.

Earlier i used to use startup disk creator with an empty ssh file into the boot drive which enabled a default user and password but that has been disabled now.

In the raspi imager ui there will be additional options for entering network details and enabling ssh which is pretty straightforward. It is neccessary to enter the network details for this headless setup as we’ll be connecting to our rpi using ssh over a wifi network so this has to be set during the image flashing process itself.

After flashing the os to the micro sd card we can put it into our rpi and boot it up.

Setting up rpi as a headless camera

For our headless setup we ssh into our rpi and set it up without a screen so for that we’ll have to use the ssh username and password that we had set while flashing the image. Along with this we connect the v2 camera module to the mipi connector.

After booting the rpi we need to set the camera settings by selecting the interface options using sudo raspi-config command

After booting the rpi we can confirm whether the camera is working or not by using the libcamera library, libcamera-hello libcamera-vid
